## Technical Decisions

### 1. Namespace Structure
**Decision:** Use relative namespaces (`Namespace Models` instead of `Namespace RCH.TemplateStorage.Models`)

**Rationale:** RootNamespace property in project file already defines `RCH.TemplateStorage`, nested namespaces caused resolution issues.

### 2. JSON Library Choice
**Decision:** Use Newtonsoft.Json instead of System.Text.Json

**Rationale:**
- More flexible serialization options
- Better handling of complex object graphs
- Easier null handling and default value configuration
- Industry standard for VB.NET projects

### 3. Legacy Import Strategy
**Decision:** Separate `ImportLegacyTemplate()` method with format auto-detection

**Rationale:**
- Clear migration path for existing templates
- No manual format specification required
- Future-proof for additional formats
- Maintains clean separation of concerns

**Solution:** Added explicit type: `Dim jObject As JObject = JObject.Parse(json)`

### Issue 4: VB.NET Lambda Syntax Confusion
**Task:** T012 (Backward Compatibility Tests)  
**Error:** `Expression does not produce a value`

**Root Cause:** Used wrong lambda syntax for predicates

**Failed Code:**
```vb
Dim folder = folders.FirstOrDefault(f => f.Name = "Test")  ' C# syntax
```

**Working Solution:**
```vb
Dim folder = folders.FirstOrDefault(Function(f) f.Name = "Test")  ' VB.NET syntax
```

**Lesson:** VB.NET uses `Function()` for lambdas that return values, `Sub()` for void lambdas

---

### Issue 5: JSON String Escaping in Test Data
**Task:** T012 (Backward Compatibility Tests)  
**Error:** `JSON was invalid` when parsing test template

**Root Cause:** VB.NET string literals don't support `@` verbatim strings like C#

**Failed Code:**
```vb
Dim json = @"{""Name"": ""Test""}"  ' C# verbatim string doesn't work in VB.NET
```

**Working Solution:**
```vb
' Use double-quotes to escape quotes in VB.NET
Dim json = "{""Name"": ""Test""}"

' Or use concatenation for readability
Dim json = "{" & vbCrLf &
          """Name"": ""Test""" & vbCrLf &
          "}"
```

**Lesson:** VB.NET string escaping differs from C# - use `""` for quotes, no verbatim strings

---
