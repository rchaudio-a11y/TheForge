#### 4. Model Validation Logic (T029)
**Validator Implementation:**
```vb
Public Class ModelValidator
    ' Validates template structure
    Public Shared Function ValidateTemplate(template As TemplateDefinition) As ValidationResult
        ' Check required fields
        If String.IsNullOrWhiteSpace(template.Name) Then
            Return Invalid("Template name is required")
        End If
        
        ' Check name length
        If template.Name.Length > 255 Then
            Return Invalid("Template name too long (max 255)")
        End If
        
        ' Validate all folders recursively
        For Each folder In template.Folders
            Dim folderResult = ValidateFolder(folder)
            If Not folderResult.IsValid Then
                Return folderResult
            End If
        Next
        
        Return Valid()
    End Function
    
    ' Validates folder structure (recursive)
    Public Shared Function ValidateFolder(folder As TemplateFolderDefinition) As ValidationResult
        ' Check required fields
        If String.IsNullOrWhiteSpace(folder.Name) Then
            Return Invalid("Folder name is required")
        End If
        
        ' Validate subfolders recursively
        For Each subFolder In folder.SubFolders
            Dim result = ValidateFolder(subFolder)
            If Not result.IsValid Then
                Return result
            End If
        Next
        
        ' Validate files
        For Each file In folder.Files
            Dim result = ValidateFile(file)
            If Not result.IsValid Then
                Return result
            End If
        Next
        
        Return Valid()
    End Function
    
    ' Validates file definition
    Public Shared Function ValidateFile(file As TemplateFileDefinition) As ValidationResult
        ' Check required fields
        If String.IsNullOrWhiteSpace(file.FileName) Then
            Return Invalid("File name is required")
        End If
        
        ' Validate encoding
        Dim validEncodings = {"UTF-8", "UTF-16", "ASCII", "UTF-32"}
        If Not validEncodings.Contains(file.Encoding) Then
            Return Invalid($"Invalid encoding: {file.Encoding}")
        End If
        
        Return Valid()
    End Function
End Class
```

**Key Features:**
- Recursive validation for unlimited folder nesting
- Comprehensive field checks
- Clear error messages
- Extensible design

## Key Architectural Decisions

### Decision 1: Property Defaults for Backward Compatibility
**Challenge:** Adding new properties might break legacy JSON deserialization.

**Solution:**
```vb
' New properties have defaults
Public Property Category As String = "General"
Public Property IsActive As Boolean = True
Public Property UsageCount As Integer = 0
Public Property CreatedDate As DateTime = DateTime.Now
```

**Result:** Legacy JSON deserializes successfully even without new fields.

---

### Decision 2: Nullable ParentFolderID for Root Folders
**Challenge:** How to represent root-level folders vs. nested folders?

**Solution:**
```vb
Public Property ParentFolderID As Integer?  ' Nullable
' Nothing = root folder
' Value = nested under parent
```

**Benefits:**
- Clear root folder identification
- Supports unlimited nesting depth
- SQL-friendly (NULL for root)

---

### Decision 3: Recursive Validation
**Challenge:** Validating deeply nested folder structures.

**Solution:** Recursive validation methods that traverse entire tree.

**Code Pattern:**
```vb
Function ValidateFolder(folder) As ValidationResult
    ' Validate this folder
    ' ...
    
    ' Recursively validate subfolders
    For Each subFolder In folder.SubFolders
        Dim result = ValidateFolder(subFolder)  ' Recursive call
        If Not result.IsValid Then Return result
    Next
    
    Return Valid()
End Function
```

**Result:** Validates folders to any nesting depth.

---

### Decision 4: IEquatable<T> Implementation
**Challenge:** Proper equality semantics for models.

**Decision:** Implement both `Object.Equals()` and `IEquatable(Of T).Equals()`

**Rationale:**
- `Object.Equals()` for general comparison
- `IEquatable(Of T)` for type-safe LINQ operations
- Both use TemplateID as primary key

---

### Challenge 2: Validation Performance
**Problem:** Recursive validation on large templates could be slow.

**Optimization:**
```vb
' Early exit on first error
For Each folder In template.Folders
    Dim result = ValidateFolder(folder)
    If Not result.IsValid Then
        Return result  ' Stop immediately
    End If
Next
```

**Result:** Validation fails fast, doesn't waste time.

---

### Challenge 3: Equality Comparison Edge Cases
**Problem:** What if TemplateID = 0 (new template)?

**Solution:**
```vb
Public Function Equals(other As TemplateDefinition) As Boolean
    If other Is Nothing Then Return False
    
    ' If both have IDs, compare IDs
    If Me.TemplateID > 0 AndAlso other.TemplateID > 0 Then
        Return Me.TemplateID = other.TemplateID
    End If
    
    ' Otherwise compare by name/description
    Return Me.Name = other.Name AndAlso
           Me.Description = other.Description
End Function
```

**Result:** Works for both persisted and new templates.

## Lessons Learned

### What Worked Well

#### 1. Default Property Values
**Approach:** All new properties have sensible defaults.

**Benefit:** Automatic migration from legacy format.

**Example:**
```vb
Public Property Category As String = "General"
Public Property IsActive As Boolean = True
```

**Result:** Legacy JSON "just works" without modification.

---

#### 2. Recursive Validation Pattern
**Approach:** Single recursive method validates entire tree.

**Code:**
```vb
Function ValidateFolder(folder) As ValidationResult
    ' Validate folder
    ' Recursively validate subfolders
    For Each sub In folder.SubFolders
        Dim result = ValidateFolder(sub)  ' Recursive
        If Not result.IsValid Then Return result
    Next
    Return Valid()
End Function
```

**Benefit:** Handles unlimited nesting elegantly.

---

#### 3. IEquatable<T> Implementation
**Approach:** Implement both `Object.Equals()` and `IEquatable(Of T)`.

**Benefit:**
- Type-safe LINQ operations
- Better performance (no boxing)
- Proper equality semantics

---

### Challenges Overcome

#### Challenge: Property Validation vs. Performance
**Issue:** Validating every property set could slow down deserialization.

**Solution:** Validate only on explicit `Validate()` call or before persistence.

**Result:** Fast deserialization, validation when needed.

---

#### Challenge: Equality for Unsaved Templates
**Issue:** New templates have TemplateID = 0.

**Solution:** Fallback to name/description comparison if no ID.

**Result:** Equality works for both new and persisted templates.

---

### Key Takeaways

1. **"Default values enable backward compatibility"**
   - Add new properties with defaults
   - Legacy data automatically valid

2. **"Validate once, save effort"**
   - Single validation point
   - Reused across service layer

3. **"Recursion handles unlimited nesting"**
   - Clean code
   - No depth limits

4. **"Test backward compatibility thoroughly"**
   - Automated tests prevent regressions
   - Confidence in changes
