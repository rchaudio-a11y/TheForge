# Source/Services/Implementations

**Purpose:** Concrete service implementations.

**Rules:**
- Contains implementation classes for interfaces
- Must implement corresponding interface from /Interfaces
- No UI references
- Services must be stateless when possible
- All dependencies must be injected

**Examples:**
- ModuleLoaderService.vb
- LoggingService.vb
- ConfigurationService.vb

**Cross-references:**
- See /Interfaces for interface definitions
- See File6.md for architecture discipline
