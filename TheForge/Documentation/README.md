# Documentation

**Purpose:** Contains all project documentation following the RCH UI Forge Documentation Taxonomy.

**Rules:**
- All documentation must be Markdown (.md)
- No code files belong here
- Follow the taxonomy defined in file4.md
- All documents must include: type, purpose, last updated date, related documents

**Structure:**
- `/Codex` — Technical references and API documentation
- `/Chronicle` — Chronological logs and version history
- `/Tomes` — User guides and tutorials
- `/Lore` — Design philosophy and naming conventions
- `/Grimoire` — Experimental and research notes
- `/Scriptorium` — Templates and generated documentation

**Cross-references:**
- See file4.md for complete documentation taxonomy
- See file1.md for Scriptorium Engine documentation generation rules
